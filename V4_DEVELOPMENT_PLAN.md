## 🚀 Phase 1：数据质量提升（3天）

> 核心短板：工具数量够了，但模型和新闻质量需要提升

### 1.1 模型列表精简
- **问题**: 294个模型中大量 GGUF/量化版本，核心模型被淹没
- **方案**: 
  - 只保留原始模型（过滤 GGUF/GPTQ/AWQ/量化版本）
  - 按 likes 排序，每个分类只显示 Top 15
  - 添加「查看全部」展开按钮
- **预期**: 294 → ~120 个核心模型

### 1.2 新闻过滤优化
- **问题**: 36氪等源混入非 AI 新闻
- **方案**:
  - 36氪源加强 AI 关键词过滤
  - 每条新闻必须包含至少一个 AI 关键词
  - 移除「护发」「汽车大灯」等无关内容
- **预期**: 新闻 100% AI 相关

### 1.3 工具数据继续扩充
- **目标**: 100 → 150+ 个工具
- **重点补充**: 教育、健康、法律、金融等垂直领域 AI 工具

## 🚀 Phase 2：搜索体验升级（2天）

> 分析报告指出搜索是核心体验，需要进一步提升

### 2.1 搜索结果关键词高亮
- 搜索词在结果中加粗/变色高亮
- 提升用户快速定位能力

### 2.2 搜索建议/自动补全
- 输入时显示下拉建议（Top 5 匹配）
- 支持键盘上下选择 + Enter 确认

### 2.3 搜索结果排序
- 默认按相关度排序
- 支持按热度/时间排序

---

## 🚀 Phase 3：详情页升级（3天）

> 分析报告指出详情页是「入口页」而非「决策页」，需大幅提升

### 3.1 添加更多详情字段
```json
{
  "pros": ["开源免费", "中文友好", "API稳定"],
  "cons": ["功能较少", "文档不全"],
  "best_for": ["个人开发者", "小团队"],
  "alternatives": ["cursor", "copilot"],
  "updated_at": "2026-04-20",
  "version": "v3.0"
}
```

### 3.2 详情页增加优缺点展示
- 绿色「✅ 优点」列表
- 红色「❌ 缺点」列表
- 帮助用户快速决策

### 3.3 详情页增加「最佳适用」标签
- 「适合个人开发者」「适合企业」「适合设计师」
- 帮助用户判断是否适合自己

### 3.4 详情页增加版本/更新信息
- 显示最后更新时间
- 显示当前版本号

---

## 🚀 Phase 4：对比功能增强（2天）

> 分析报告评分 4/5，仍有提升空间

### 4.1 对比维度扩展
- 增加「优点」「缺点」对比行
- 增加「适用场景」对比行
- 增加「更新频率」对比行

### 4.2 支持模型对比
- 新增模型对比模式
- 对比维度：参数量、likes、pipeline_tag、作者

### 4.3 工具卡片添加「加入对比」按钮
- 在工具/模型卡片上添加「+ 对比」按钮
- 点击后加入对比列表，最多 4 个

---

## 🚀 Phase 5：上瘾机制深化（2天）

> 分析报告指出上瘾机制「基础框架在，但不够丰富」

### 5.1 新收录板块改进
- 显示最近 7 天内新增的所有工具（不限 trending）
- 标记「🆕 新」标签
- 显示新增时间（如「3天前收录」）

### 5.2 每日签到
- 首页添加「签到」按钮
- 签到获得连续访问天数 +1
- 签到后显示今日数据变化摘要

### 5.3 分享功能
- 添加「分享到微信/微博」按钮
- 生成分享卡片（带二维码）
- 支持分享工具详情、对比结果

## 🚀 Phase 6：SEO 深度优化（1天）

### 6.1 子页面独立 title
- 工具页: `Cursor - AI代码编辑器 | AI热榜`
- 模型页: `Llama-3-8B - Meta开源大模型 | AI热榜`

### 6.2 结构化数据 (Schema.org)
- 工具页添加 `SoftwareApplication` schema
- 新闻页添加 `NewsArticle` schema

### 6.3 sitemap 自动生成
- GitHub Actions 每次更新时自动生成 sitemap.xml
- 包含所有工具详情页链接
