Meta 内部最近流行起一种很硅谷的新攀比方式：不是比谁写了多少代码，也不是比谁做了多少实验，而是比谁一个月“烧”掉了更多 Token。根据 The Information 报道，Meta 员工自发做了一个名为“Claudeonomics”的内部排行榜，用来统计超过 8.5 万名员工近 30 天的模型调用量。

这件事之所以值得看，不只是数字夸张，而是它把一个原本属于后台预算表的指标，推成了组织文化的一部分。过去企业谈 AI，更多还是讨论模型效果、产品能力和落地场景；现在在一线团队里，Token 消耗本身已经开始被包装成一种“你到底会不会用 AI”的证明。

报道里提到，Meta 过去 30 天的 Token 总消耗据称超过 60 万亿。哪怕考虑到企业采购价格与公开定价不同，这仍然意味着非常惊人的计算投入。更夸张的是，内部个人用量最高者达到 2810 亿 Token，足以说明大公司里一部分工程师已经把模型调用当成基础设施来使用，而不是偶尔点开的聊天工具。

这种变化和 Agent 工作流直接相关。传统对话式 AI 的成本还比较容易感知：提一个问题，得到一个回答，账单大致能算清。但当越来越多任务被交给能长时间运行、会自主拆解步骤、会反复调用工具的 Agent 系统后，Token 消耗会转移到用户看不见的后台流程里。一句“你自己去做完”背后，可能是几十轮推理、上百次调用和持续不断的上下文读写。

所以 Meta 的 Token 榜既是 AI 深入组织的信号，也暴露出新的管理问题：企业到底应该鼓励“多用模型”，还是要求“用得值”？如果组织只奖励 Token 用量，团队很容易把资源消耗误当成效率提升本身，最后形成新的 KPI 游戏。

更现实的一点是，硅谷高管正在公开强化这种文化。无论是把高额 Token 开销视为优秀工程师的必要投入，还是把“用不完 Token 会焦虑”当成行业态度，本质上都在推动一个共识：AI 时代的核心竞争力，不只是人力，而是你敢不敢持续调动足够多的算力。

但这套逻辑能否长期成立，还要看另一件事：那些不可见的 Token 流水，到底换回了多少真正有价值的结果。对企业来说，下一阶段的难题恐怕不再是“要不要给团队开放模型额度”，而是如何衡量一条 Agent 链路到底创造了多少真实产出。
