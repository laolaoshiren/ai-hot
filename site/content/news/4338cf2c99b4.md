OpenAI 发布 GPT-5.5 本来应该是一场稳稳占据话题中心的更新：更强的编程表现、更偏向智能体执行的定位，以及明显拉高的 API 定价。但几乎没过一天，DeepSeek 就把 V4 权重推上 Hugging Face，又用远低于头部闭源模型的价格，把开发者的注意力迅速拉走。

这篇文章最有意思的地方，不是简单比较跑分，而是呈现了真实用户的迁移动作。一位重度 AI 编程用户直接表示，自己把 Claude Code、Codex、Cursor、Aider 等工作流都改接 DeepSeek 原生接口，理由非常直接：账单可能下降 90%，而效果没有变差，甚至更好。对普通围观者来说，这像一条情绪化社媒发言；但对行业来说，它反映的是默认模型正在重新洗牌。

价格差距确实足够夸张。按照文中给出的口径，GPT-5.5 标准版每百万输入和输出 token 合计大约 35 美元，Claude Opus 4.7 约 30 美元，而 DeepSeek-V4-Pro 只要 5 美元出头，命中缓存后还能再降。更激进的 V4-Flash 则把价格进一步压进“几乎可以忽略”的区间。对于高频编码、长上下文代理和企业批量调用场景，这种差距不是财务细节，而是产品决策本身。

更关键的是，DeepSeek 这次提供的是 MIT 协议开源权重。也就是说，它不只是用低价 API 去卷 OpenAI 和 Anthropic，还顺带抢占了本地部署、数据出域敏感和私有化合规这类闭源模型更难覆盖的市场。对于已经把 AI 编程接入研发流程的团队来说，这会显著改变采购和架构讨论。

这轮变化说明，编程模型市场已经进入更现实的阶段。开发者不再只关心谁的 benchmark 更亮眼，而是在问：同样的任务，谁能更稳定地跑完、谁的账单更可控、谁更适合进正式工作流。DeepSeek V4 之所以值得看，不只是因为它便宜，而是因为它让“高性能 + 可负担”第一次真正变成了一组难以忽视的组合。
